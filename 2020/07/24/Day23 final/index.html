final

  1. 昨天学习了一天的final，意思是‘最终’的意思。
  2. 它是一个修饰符，用来修饰变量之类的。
  3. final修饰实例变量后要手动赋值，不然编译报错。
  4. final修饰的变量只能赋值一次。
  5. final static 修饰的变量为常量。

常量名全部大写

常量名规范是全部大写，当然你全部小写也可以通过，这只是规范，说严重点就是防君子不防小人咯。

  1. final修饰的引用，一旦指向一个对象，则不可以重新指向别的的对象。

final修饰的类无法继承

final修饰的方法无法覆盖

  方法覆盖：在两个具有继承关系的类中，子类中有一个方法，如果这个方法的返回值类型，方法名，参数列表与父类中的一个方法一摸一样，那么我们说这个父类的这个方法被覆盖了。 如果这个子类中的方法被你修改了，那就叫方法重写。

final修饰的实例变量必须手动赋值

不能采用系统赋默认值。

class A{
    final int i;  //编译错误，变量i未初始化  
    final int 1 = 1;//编译通过
}

  就是你变量i = 100；<—— i = 100就是变量，它可以多次赋值。但是你加上final static之后 ——> final static i = 100;他i就只能是100，你再赋值你赋不上去，编译器会报错。一般用来形容一些固定不会变的东西吧，这就是常量。
